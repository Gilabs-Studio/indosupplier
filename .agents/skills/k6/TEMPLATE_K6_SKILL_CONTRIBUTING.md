# Contributing to the K6 Skill

## Guidelines
- Follow clean architecture and best practices in all templates and docs
- Keep code, documentation, and templates in sync
- Update the coverage matrix and relationship diagrams after each change
- Use Gill Me mode for all new workflow implementations
- Review and test generated scripts and docs
- Document all changes in the changelog

## How to Propose Changes
- Fork the repo and create a feature branch
- Update relevant templates, docs, and the changelog
- Submit a pull request with a clear description
