# K6 Skill Security Guide

## Security Best Practices
- Never commit secrets or credentials in scripts or docs
- Use environment variables for sensitive data
- Validate all API responses and error handling
- Review scripts for injection or concurrency risks
- Keep dependencies up to date

## How to Use
- Follow security best practices in all workflow implementations
- Review generated scripts for potential vulnerabilities
- Report security issues to maintainers

---
*Security is a shared responsibility. Follow this guide to keep k6 tests safe.*
