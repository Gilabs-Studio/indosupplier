# K6 Skill Best Practices

- Always use authentication and CSRF handling helpers
- Prefer dynamic data and randomized pools for test realism
- Model full transaction cycles, not just single endpoints
- Explicitly document and test inter-module dependencies
- Use Gill Me mode to clarify business context and edge cases
- Keep documentation and code templates in sync with backend contracts
- Update the coverage matrix after every new workflow implementation
- Review generated docs/scripts for accuracy and completeness
- Follow clean architecture layering in documentation: use-cases, infrastructure, architecture
- Regularly maintain and refactor skill templates as business needs evolve

---
*Follow these best practices for every k6 workflow implementation and documentation session.*
